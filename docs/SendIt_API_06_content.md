# CONVENTIONS
AUTH: Bearer JWT in Authorization header on all endpoints except public auth routes.
OWNERSHIP: All user endpoints auto-filter by authenticated user ID.
REST: GET list, GET /:id, POST create, PUT /:id update, DELETE /:id delete — unless noted.
ERRORS: {"statusCode":N,"error":"CODE","message":"..."} 

# MODULE: Templates
Reusable shipment form presets. Never submitted to operators. Stored locally in SendIt DB.
usageCount incremented via PUT /templates/:id when shipment created from template.

GET /templates
  params: operator(slug), shipmentType(DOCUMENT|PACKAGE|BOX|CARGO|PALLET), search(name+desc), sortBy(createdAt|name), sortOrder, page, limit
  OUT 200: {data:[{id,name,description,postalService:{name,slug},shipmentType,usageCount,createdAt}],total,page,limit}

GET /templates/:id
  OUT 200: full template with templateData (all form fields)
  ERR: 404, 403 if not owner

POST /templates
  IN: {name, description?, postalServiceId, shipmentType, templateData:{...}}
  NOTE: also used for duplication — send POST with source data and modified name ("Copy of X")
  OUT 201: created template

PUT /templates/:id
  IN: any subset of template fields
  NOTE: also used to increment usageCount when shipment created from this template
  OUT 200: updated template

DELETE /templates/:id
  LOGIC: hard delete. Linked shipments unaffected (template reference unlinked).
  OUT 204

SHIPMENT_TYPES: DOCUMENT | PACKAGE | BOX | CARGO | PALLET | UNKNOWN
Status mapper pattern same as shipment statuses — hardcoded per operator.

# MODULE: Recipients
Address book of saved contacts (organizations or individuals). Optional delivery address per contact.

GET /recipients
  params: type(ORGANIZATION|INDIVIDUAL), search(name+phone+email), sortBy, sortOrder, page, limit
  OUT 200: {data:[{id,type,companyName?,ownershipForm?,edrpou?,lastName,firstName,patronymic?,phone,email?,note?,address?}],total,page,limit}
  address shape: {type:BRANCH|ADDRESS|PARCEL_LOCKER|DIGITAL, city?, branchNumber?, street?, building?, apartment?, comment?, lockerNumber?, identifier?}

GET /recipients/:id
  OUT 200: full recipient
  ERR: 404, 403

POST /recipients
  IN: {type, companyName?(org), ownershipForm?, edrpou?, lastName?(ind), firstName, patronymic?, phone, email?, note?, address?}
  address is optional — one address max per recipient
  OUT 201: created recipient

PUT /recipients/:id
  IN: any subset of recipient fields including address
  OUT 200: updated recipient

DELETE /recipients/:id
  LOGIC: hard delete. Linked shipments unaffected.
  OUT 204

# MODULE: Notifications
In-app notifications generated by SendIt backend only (no webhooks from operators in current scope).

TYPES: SUBSCRIPTION | POSTAL_CONNECTION | ACCOUNT | SYSTEM | SHIPMENT_STATUS

LIFECYCLE:
  isRead:false → never auto-deleted
  isRead:true → auto-deleted after 30 days (cron)
  Manual delete → immediate

GET /notifications
  params: tab(unread|all, default:all), type(SUBSCRIPTION|POSTAL_CONNECTION|ACCOUNT|SYSTEM|SHIPMENT_STATUS), page, limit
  OUT 200: {items:[{id,type,title,body,isRead,createdAt,updatedAt}],meta:{page,pageSize,totalItems,totalPages,hasNextPage,hasPreviousPage}}

GET /notifications/unread-count
  NOTE: polled by frontend every 60s for header badge
  OUT 200: {count:N}

PUT /notifications/:id
  IN: {isRead:true}
  OUT 200: updated notification

PUT /notifications
  LOGIC: mark ALL unread as read for current user
  OUT 200: {updated:N}

DELETE /notifications/:id
  OUT 204

DELETE /notifications
  params: filter(read|none)
    filter=read → delete all isRead:true
    no filter → delete ALL for current user (confirmation required on frontend)
  OUT 204

INTERNAL CREATION (called by other modules, not by client):
  createNotification(userId, type, title, body)
  RESPECTS PREFERENCES: subscription/postalConnection/system can be disabled by user settings; account notifications are mandatory
  Callers: SubscriptionService, PostalConnectionsService, ProfileService(account), shipment sync, AdminService(system broadcast)

CRON: cleanupReadNotifications — daily 02:00
  DELETE FROM notifications WHERE isRead=true AND updatedAt < now()-30d

# MODULE: Onboarding
Lightweight read-only checklist for new users. Computed server-side from existing records.

GET /onboarding/checklist
  LOGIC:
    profileCompleted: User.profileCompleted === true
    operatorConnected: count(ACTIVE UserPostalConnection) > 0
    firstShipmentCreated: count(shipments from operators + drafts) > 0
  OUT 200: {profileCompleted:bool, operatorConnected:bool, firstShipmentCreated:bool}
